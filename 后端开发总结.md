## 数据库模型

### User（用户表）
- id: UUID 主键
- username: 用户名（唯一）
- email: 邮箱（唯一）
- password: 加密密码
- avatar: 头像（可选）
- height: 身高（可选）
- weightGoal: 目标体重（可选）
- createdAt, updatedAt: 时间戳

### Activity（运动记录表）
- id: UUID 主键
- userId: 关联用户
- type: 运动类型
- duration: 时长（分钟）
- calories: 消耗卡路里
- distance: 距离（公里，可选）
- notes: 备注（可选）
- date: 日期

### DietRecord（饮食记录表）
- id: UUID 主键
- userId: 关联用户
- mealType: 餐次类型
- items: 食物列表（JSON）
- totalCalories: 总卡路里
- totalProtein: 总蛋白质
- totalCarbs: 总碳水化合物
- totalFat: 总脂肪
- date: 日期

### Food（食物表）
- id: UUID 主键
- name: 食物名称
- caloriesPer100g: 每100克卡路里
- proteinPer100g: 每100克蛋白质
- carbsPer100g: 每100克碳水化合物
- fatPer100g: 每100克脂肪
- category: 分类
- isCustom: 是否自定义
- userId: 创建用户（自定义食物时）

### WeightRecord（体重记录表）
- id: UUID 主键
- userId: 关联用户
- weight: 体重（公斤）
- bodyFat: 体脂率（可选）
- muscleMass: 肌肉量（可选）
- date: 日期

## 快速开始

### 1. 安装依赖

```bash
cd backend
npm install
```

### 2. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env，填入数据库连接信息和 JWT 密钥
```

### 3. 初始化数据库

```bash
npm run db:generate   # 生成 Prisma Client
npm run db:push       # 同步数据库结构
```

### 4. 启动开发服务器

```bash
npm run dev
```

服务器将在 http://localhost:3000 启动

## 部署准备

### 前端适配

要使用后端 API，前端需要：

1. 创建 API 客户端（使用 axios 或 fetch）
2. 实现登录/注册页面
3. 重构数据存储，从 localStorage 改为调用 API
4. 处理 JWT 令牌的存储和刷新
5. 更新所有数据操作接口

### 部署选项

**最小成本方案：**
- 前端: Vercel 免费版

**生产环境方案：**
