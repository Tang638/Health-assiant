
# 智能健康助手项目 - 部署上线准备清单

## 当前项目状态分析

当前项目是一个**纯前端个人项目**，包含：
- ✅ React 18 + TypeScript + Vite 前端框架
- ✅ 完整的页面功能（运动、饮食、体重等）
- ✅ 使用 localStorage 本地存储
- ❌ 没有后端服务
- ❌ 没有数据库
- ❌ 没有用户注册功能
- ❌ 没有真实的认证系统
- ❌ 没有数据持久化（用户数据在浏览器本地）

## 第二阶段：安全加固（必须）

### 2.1 认证安全

1. **密码加密**
   - 使用 bcrypt 或 argon2 加密密码
   - 最小密码长度 8 位
   - 密码复杂度要求（大小写、数字、特殊字符）

2. **JWT 安全配置**
   ```javascript
   // 后端配置示例
   accessToken: {
     secret: process.env.JWT_SECRET,
     expiresIn: '15m',
   },
   refreshToken: {
     secret: process.env.JWT_REFRESH_SECRET,
     expiresIn: '7d',
   },
   ```

3. **刷新令牌机制**
   - 使用访问令牌 + 刷新令牌双令牌机制
   - 刷新令牌存储在 HTTP-only Cookie 中

### 2.2 接口安全

1. **CORS 配置**
   ```javascript
   const corsOptions = {
     origin: process.env.FRONTEND_URL,
     credentials: true,
     methods: ['GET', 'POST', 'PUT', 'DELETE'],
   };
   ```

2. **速率限制**
   - 防止暴力破解：登录接口限制 5次/分钟
   - API 调用限制：100次/分钟

3. **数据验证**
   - 使用 Zod 或 Joi 进行输入验证
   - 防止 SQL 注入（使用参数化查询）
   - 防止 XSS（前端 + 后端双重过滤）

### 2.3 安全头设置

在部署时设置以下 HTTP 响应头：
```
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
Content-Security-Policy: default-src 'self'
Strict-Transport-Security: max-age=31536000
```

## 第三阶段：前端适配（必须）

### 3.1 API 集成

将当前的 localStorage 改为调用后端 API：

1. **创建 API 客户端**
   ```typescript
   // src/api/client.ts
   import axios from 'axios';
   
   const api = axios.create({
     baseURL: import.meta.env.VITE_API_URL,
     timeout: 10000,
     withCredentials: true,
   });
   
   // 自动添加 JWT token
   api.interceptors.request.use((config) => {
     const token = localStorage.getItem('accessToken');
     if (token) {
       config.headers.Authorization = `Bearer ${token}`;
     }
     return config;
   });
   
   // 处理 401 未授权
   api.interceptors.response.use(
     (response) => response,
     async (error) => {
       if (error.response?.status === 401) {
         // 尝试刷新 token 或跳转到登录
         window.location.href = '/login';
       }
       return Promise.reject(error);
     }
   );
   
   export default api;
   ```

2. **重构 store**
   - 将 zustand store 改为从 API 获取数据
   - 添加加载状态和错误处理

### 3.2 添加用户注册功能

#### Docker 部署（推荐）

创建 `Dockerfile`：
```dockerfile
# Dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:18-alpine AS runner
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY package.json ./
ENV NODE_ENV=production
EXPOSE 3000
CMD ["node", "dist/main.js"]
```

创建 `docker-compose.yml`：
```yaml
version: '3.8'
services:
  app:
    build: .
    ports:
      - '3000:3000'
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/health
      - JWT_SECRET=your-secret-key
    depends_on:
      - db
  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=pass
      - POSTGRES_DB=health
    volumes:
      - postgres_data:/var/lib/postgresql/data
volumes:
  postgres_data:
```
